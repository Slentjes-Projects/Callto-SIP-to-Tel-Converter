Callto/SIP to Tel Converter - Documentation

Extension Information

Name: Callto/SIP to Tel Converter
Version: 1.1
Purpose: Automatically converts callto: and sip: links to tel: links
Supported browsers: Microsoft Edge, Google Chrome
Operating system: Windows, macOS, Linux

What is this extension?

The Callto/SIP to Tel Converter is a browser extension that automatically converts phone number links on websites. When a website uses a link with callto: or sip: protocol, this extension converts these links to tel: links. This ensures that users can dial phone numbers via their phone system (such as Microsoft Teams, 3CX, or other VoIP systems) instead of only through the browser.

Who is this documentation for?
This documentation is intended for first-line support (helpdesk). It enables you to help users with installation, management, and common issues with the extension.

Common Problems and Quick Solutions

Problem 1: The extension doesn't work after installation

Symptoms: Clicking phone numbers still opens the browser dialer instead of Teams/3CX
Quick diagnosis:
  1. Check if the extension is enabled in Edge/Chrome settings
  2. Refresh the website where you're testing (F5 or Ctrl+R)
  3. Test on another website with callto/sip links
Solution:
  1. Open edge://extensions (Edge) or chrome://extensions (Chrome)
  2. Look for "Callto/SIP to Tel Converter"
  3. Make sure the toggle is BLUE (enabled)
  4. Refresh the website (Ctrl+F5)
  5. Test again

Problem 2: The extension doesn't work on some websites

Symptoms: The extension works on website A but not on website B
Possible causes:
  • Website B creates links dynamically (AJAX/JavaScript)
  • Website B blocks extensions
  • Website B uses different link formats
Solution:
  1. Check if the link actually starts with callto: or sip:
  2. Right-click on the link → "Inspect" (or F12)
  3. Look for href="callto:..." or href="sip:..."
  4. If this is NOT present, the extension won't work (this is a website issue, not an extension issue)
  5. If this IS present, reloading the page may help

Problem 3: User gets error "This app cannot be opened"

Symptoms: After clicking a phone number, an error message appears
Possible causes:
  • Microsoft Teams/3CX is not installed
  • The default phone application is not configured
  • The link is corrupted
Solution:
  1. Check if Microsoft Teams or 3CX is installed and active
  2. In Teams: Check if Teams is set as the default VoIP app
  3. Remove any spaces or special characters from the phone number
  4. Test with a different phone number

Problem 4: The extension disappears after browser restart

Symptoms: After closing and reopening the browser, the extension is gone
Cause: This happens when the extension is loaded in "Developer Mode" without saving it in permanent settings
Solution:
  1. Reload the extension from the file again
  2. Contact IT for permanent installation options
  3. In the future, the extension will be available via Chrome Web Store / Edge Add-ons

Problem 5: User doesn't see the extension in the browser list

Symptoms: Cannot find "Callto/SIP to Tel Converter" in edge://extensions
Possible causes:
  • Extension is not correctly loaded
  • User is looking in the wrong place
  • Developer Mode is disabled
Solution:
  1. Make sure Developer Mode is enabled (see step 2 in installation guide)
  2. Check the correct path: edge://extensions (Edge) or chrome://extensions (Chrome)
  3. Scroll down - extension may be at the bottom
  4. Ask the user to take screenshots for verification

Installation Guide: Step by Step

For Microsoft Edge

Step 1: Open the Extensions page
  • Click on the menu (three dots) in the top right of Edge
  • Select "Extensions" → "Manage extensions"
  • Or type directly in the address bar: edge://extensions
Step 2: Enable Developer Mode
  • Look at the bottom left of the page
  • Toggle "Developer mode" to ON (switch turns blue)
Step 3: Load the extension
  • Click the "Load unpacked" button (top left)
  • Navigate to the extension folder
  • Click "Select Folder"
Step 4: Confirm installation
  • You should now see "Callto/SIP to Tel Converter" in the list
  • The toggle is blue (enabled)
  • Version number shows "1.1"
Step 5: Test the extension
  • Open a website with phone numbers
  • Right-click on a phone number and choose "Inspect"
  • Check if href="tel:+31612345678" is displayed (not callto: or sip:)
  • Try clicking the phone number

For Google Chrome

Step 1: Open the Extensions page
  • Click on the menu (three dots) in the top right of Chrome
  • Select "More tools" → "Extensions"
  • Or type directly in the address bar: chrome://extensions
Step 2: Enable Developer Mode
  • Look at the top right of the page
  • Toggle "Developer mode" to ON (switch turns blue)
Step 3: Load the extension
  • Click "Load unpacked"
  • Navigate to the extension folder
  • Select the folder and click "Select Folder"
Step 4: Confirm installation
  • You should now see "Callto/SIP to Tel Converter" in the list
  • The toggle is blue (enabled)
  • Version number shows "1.1"
Step 5: Test the extension
  • Open a website with phone numbers

Frequently Asked Questions (FAQ)

Q: Does this extension work on mobile phones?

A: No, this extension is only for browsers on computers (Edge and Chrome). Mobile browsers already have native tel: support.

Q: Will this extension slow down my browser?

A: No. The extension works minimally (only link conversion) and has no noticeable impact on browser performance.

Q: What happens to my privacy and data?

A: The extension processes no data, logs nothing, and sends nothing to external servers. It works entirely locally in your browser.

Q: Can I use the extension on all websites?

A: Yes, the extension works on all websites, except when:

  • The website blocks the extension
  • The website has strict Content Security Policy (CSP)
  • The website doesn't use callto: or sip: links

Q: What if I accidentally delete an extension file?

A: Contact IT. The extension can be reinstalled from the central location.

Q: Does this extension work with Microsoft Teams and 3CX?

A: Yes! This is exactly the purpose. After links are converted to tel:, they automatically open in:

  • Microsoft Teams (if set as default VoIP app)
  • 3CX (if set as default VoIP app)
  • Other VoIP systems that support tel: protocol

Q: What happens if both Edge and Chrome are on the same computer?

A: Both can have the extension. They work independently of each other. Install the extension for both browsers if needed.

In Microsoft Edge:
  1. Open edge://extensions

In Google Chrome:

Escalation Procedure

When NOT to escalate to Second Line

You can solve these problems yourself:

  • Extension not enabled (check toggle)
  • Website not refreshed (Ctrl+F5)
  • Wrong installation path followed (repeat installation guide)
  • User tested on website without callto:/sip: links
  • VoIP app (Teams/3CX) not set as default

When to escalate to Second Line

Escalate in these scenarios:

Scenario 1: Extension crashes browser

Symptoms: Browser crashes or becomes slow
Action: Remove extension, report bug

Scenario 2: VoIP system doesn't respond to tel: links

Symptoms: Links work after conversion, but Teams/3CX doesn't open
Required info:
  • VoIP system name and version
  • Screenshot of Developer Tools console
  • Working phone number that was tested

Scenario 3: Extension doesn't work on certain company websites

Symptoms: All other websites work, but internal website doesn't
Required info:
  • Exact URL of website
  • Actual link format (screenshot Developer Tools)
  • CSP error messages from console

Scenario 4: Installation/Removal Issues

Symptoms: Cannot install or remove extension
Required info:
  • Exact error message
  • Screenshot of extensions page

Scenario 5: User reports security issue

Symptoms: Suspicious behavior, unexpected network activity
Action: IMMEDIATELY escalate, remove extension

Technical Details (For IT Administrators)

How does the extension work?

  1. Initialization: The extension loads when the browser starts
  2. Scanning: On each page load, it scans for links with callto: and sip: protocol
  3. Conversion: These links are replaced with tel: links
  4. Monitoring: The extension monitors dynamically loaded content (AJAX/SPAs) and converts these links too

Known Limitations

  • Dynamic content: Extension follows DOM mutations, but not everything
  • PDFs: Doesn't work in embedded PDFs (browsers block this)
  • Chrome Extensions API: Only works in Chrome/Edge, not in Firefox
  • CSP-blocking: Some company sites block extensions
